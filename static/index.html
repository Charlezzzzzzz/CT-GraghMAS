    日常通用模式

    你可以把平时遇到的小问题放在这里。系统会按设置页里的学习者画像检索图谱和相关知识，只给你必要提示、概念澄清和下一步行动，不展开完整课堂流程。

    低干扰相关知识 概念澄清 最小样例 不强制套课堂流程
    日常答疑就绪

    你可以问“为什么报错”“这个概念是什么意思”“下一步先检查哪里”。需要按课堂流程推进时，再切换到教学伴学页。

    教学伴学参数

    日常：相关知识 + 必要提示 教学：流程调度 + 智能体分工 ADAPT：诊断-支架-行动-练习-迁移 C-O-D-E-R：情境-编排-开发-调试-重构 图谱：知识点-先修-错误-相关知识
    教学模式会把 C-O-D-E-R/ADAPT 映射到 L1-L2-L3 调度链。
    教学伴学就绪

    你可以把课堂任务、项目活动或练习卡点放在这里。系统会按你在设置页选择的画像和流程，把 ADAPT 或 C-O-D-E-R 映射到 L1-L2-L3 智能体调度。

    用户与角色

    系统边界

    反代写门控你可以把完整代码请求降级为提示、伪代码和局部检查。
    相关知识边界你可以让系统只围绕当前路由节点扩展必要关系，避免上下文过载。
    语料注入你可以只启用少量高相关知识，让反馈保持清晰和可解释。
    教师可监督你可以复盘和校准交互记录、图谱相关知识与课堂支架。

    System Settings

    设置页面

    你可以在这里统一配置学习者画像、多智能体编排、API 接入、输出边界和课堂运行偏好；系统会按你当前的身份，把这些设置落到对应的学习、教学或管理流程里。

    Agent Orchestration

    多智能体配置

    你可以配置学习者画像、默认工作模式、路由分配策略、教学流程、表达风格和参与协同的 L2 智能体。系统动态分配会依据 L1 诊断和路由结论自动选择 P1/P2/P3/P4。

    配置会保存在当前浏览器，用于本地演示与论文系统展示。

    API Configuration

    模型 API 设置

    你可以在这里配置 OpenAI-compatible、本地模型或学校自建网关。当前原型仍可在无 API 情况下运行；填写后可作为后续替换 L2 智能体和 embedding 模型的统一入口。

    设置仅保存在本地 SQLite 中。

    接入位置

    L2 智能体生成你可以把 P1/P2/P3/P4 的规则模板逐步替换为真实模型调用。
    语料 Embedding你可以把当前 hashing embedding 替换为配置的 embedding 模型。
    教学门控你需要让外部模型输出继续经过反代写、认知适配和教师监督边界。
    图谱增强你可以要求模型回答携带当前路由相关知识、图谱节点或教师配置的语料来源。

    Prompt Workflow

    核心提示词与工作流

    这里把测试脚本中的 L1-L2-L3 提示词、145 节点意图映射、2-hop 学情抽取和动态路由流程写入系统。你可以在配置 API 后逐步把本地演示智能体替换为真实模型调用。
